## Objective

Unified job provider interface with normalized ingestion, source health, NOT_CONFIGURED states for key-based connectors, fixtures, and ad hoc runs.

## Delivered

- `job_providers.py` — provider registry (Greenhouse, Lever, Ashby, Remotive, Remote OK, RSS, Adzuna, Jooble, USAJOBS, fixture)
- `connector_runner.py` — run orchestration with audit trail
- `/api/connectors` — list status, run live/fixture
- Job Connectors UI at `/connectors`
- Config env vars: `ADZUNA_*`, `JOOBLE_API_KEY`, `USAJOBS_*`, `RSS_FEED_URLS`
- Tests: `tests/test_job_connectors.py` (7 scenarios)

## Architecture changes

- Extended `config/source-policy.yml` with new allowed automated sources
- Key-based connectors return `NOT_CONFIGURED` without API keys; fixture runs always available

## Migrations

None.

## Known limitations

- Scheduled connector runs deferred (n8n workflows remain disabled locally)
- RSS requires `RSS_FEED_URLS` configuration
- Live Remote OK / Remotive subject to upstream rate limits
- Connector last-run state is in-memory (resets on API restart)
