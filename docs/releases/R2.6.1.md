# R2.6.1 — Auth session and Google persistence patch

**Type:** Patch on r2.6.0  
**Tag:** `r2.6.1`

## Why

After container or stack restarts, the UI could render the authenticated shell while API calls failed because the browser kept a stale JWT in `localStorage` with no server-side session validation.

## What changed

- Database-backed sessions (`auth_sessions`) with HttpOnly `careeros_session` cookie
- `GET /api/auth/session` gate; splash screen until validated
- Login page with Remember Me; session-expired messaging; nav logout
- All web pages migrated from `localStorage` bearer tokens to cookie `apiFetch`
- Google OAuth: preserve refresh tokens, `connection_status`, `last_refresh_at`, reconnect-only consent
- Settings UI: Connected / Reconnect / Disconnected states without exposing tokens
- Docker path resolution fix for `/app` layout in `config.py` and `conftest.py`

## Migration

- Alembic `0007_auth_sessions` — run automatically on API startup
