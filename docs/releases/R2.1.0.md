# R2.1.0 — Application Ledger and Duplicate Protection

Date: 2026-06-28  
Tag: `r2.1.0`  
Commit: `6a48438`  
Prior tag: `r2.0.0` (`98dae5921668e8b2fed5a8aa84835c61d28943c8`)

## Objective

Add canonical company registry, application ledger, duplicate-risk engine, resume factual-core validation, UI indicators, and autonomous-mode backend rejection before broad job ingestion.

## Delivered

- Migration `0004_duplicate_protection`: companies, aliases, domains, ATS identities, application ledger/events, duplicate overrides; job company linkage fields
- Services: `duplicate_risk.py`, `factual_core.py`
- API router: `/api/companies` (ledger, duplicate-risk, override, application modes)
- Integration: duplicate checks on packet generation and approval; ledger updates on packet/approval
- UI: Company Ledger page, duplicate-risk indicator on job detail, application modes in Settings
- Tests: `tests/test_duplicate_risk.py` (8 scenarios)

## Architecture changes

- Jobs link to canonical `companies` on ingest
- Application ledger records status transitions with audit events
- RED duplicate risk blocks packet generation and approve/submit actions until override
