# R2.5.0 — Manual Application Workflow

**Tag:** `r2.5.0`  
**Release date:** 2026-06-28  
**Branch:** `main`

## Summary

First complete supervised manual application workflow: job review with fit/trust/duplicate/vendor signals, packet generation with validation, human approval, immutable document versions, official Apply link with pre-open checks, user-recorded outcomes, and chronological timeline.

## Deliverables

### A. PostgreSQL duplicate enforcement

- Global exact-match checks for application URL, ATS job ID, source+external ID, and company-scoped requisition ID before company-only heuristics.
- `POLICY_VERSION` bumped to `r2.5.0`.
- Ledger records store `source` and `external_id` on create.
- Integration tests in `tests/test_duplicate_risk_postgres.py` (7 scenarios; run when `DATABASE_URL` is PostgreSQL).

### B. Vendor/client representation conflict

- `representation_records`, `representation_overrides` tables and `/api/representations` CRUD.
- Job-level `representation-risk` and override endpoints with audit trail.
- Policy: active vendor+client+requisition → RED; expired → AMBER; override with reason → GREEN.
- Integrated into packet generation and approval gates.

### C. Immutable document versions

- `application_document_versions` table with concurrency-safe version allocator (`SELECT … FOR UPDATE`).
- Versioned paths under `job_{id}/v{nn}/`.
- Submitted versions marked immutable; regeneration creates v02+ without overwriting v01 files.
- Timeline events for workflow steps.

### D. Lever connector hardening

- Demo slug `leverdemo` with test-only fixture mode (no live HTTP).
- Structured `NO_ACTIVE_BOARD` errors for 404 and non-list responses.
- Configurable `LEVER_API_BASE` for global/EU endpoints.
- Per-board failure isolation in connector runner.
- Runbook: `docs/runbooks/LEVER_CONNECTOR.md`.

### E. Manual workflow UX

- Job detail: plain-English fit/trust, duplicate and vendor panels, apply readiness.
- Applications page: validation status, approve/reject/submit flow with confirmation, version history, timeline.
- Approval queue retained for packet preview/download.

### F. CI and Playwright

- CI fix for `/app/generated` path on GitHub Actions (commit `744d79f`).
- New `playwright-fixture` job: API + web in fixture mode, no production keys.
- Expanded `r25-workflow.spec.ts` journeys (login, duplicate block, vendor warning, apply readiness, autonomous 403).

## Known limitations

- Assisted apply mode remains disabled until R2.6.
- Autonomous submission returns HTTP 403.
- Full R2.10 UI redesign deferred.
- Live Google Drive end-to-end proof requires owner browser OAuth step.
- Description-fingerprint RED is company-scoped (same company + role + fingerprint).
