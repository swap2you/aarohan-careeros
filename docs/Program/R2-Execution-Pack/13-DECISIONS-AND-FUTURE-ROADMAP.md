# Decisions and Future Roadmap

## Approved now

- local-first private application,
- direct commits to main,
- release tags and release notes,
- legitimate API/feed/ATS sources,
- email-alert ingestion for restricted job boards,
- manual and assisted application modes,
- final external submission remains human-controlled,
- OpenAI for document intelligence and TTS,
- Ask Aarohan over internal records,
- full local engineering authority with audit/backup controls,
- duplicate-application ledger and resume factual consistency,
- professional modern UI,
- future cloud/SaaS path documented but not deployed.

## Explicitly not approved in R2

- autonomous job submission,
- CAPTCHA bypass,
- LinkedIn/Indeed/Glassdoor scraping,
- invented profile facts,
- commercial SaaS launch,
- multi-tenant production deployment,
- paid data subscriptions without measured need.

## Future SaaS roadmap

### R3 — Cloud pilot

- cloud hosting,
- secure auth,
- private single-user environment,
- encrypted storage,
- production scheduler/worker,
- monitoring,
- backups,
- privacy/export/delete.

### R4 — Multi-user beta

- tenant isolation,
- onboarding,
- plans/limits,
- billing integration,
- support and admin tooling,
- legal review,
- terms/privacy/AI disclosures.

### R5 — Commercial product

- licensed job data,
- source partnerships,
- enterprise security,
- compliance review,
- product analytics,
- subscription packages,
- customer support,
- audited autonomous-workflow research.

No future release should be implemented merely because it is listed. It requires an explicit decision after R2 usage data.
