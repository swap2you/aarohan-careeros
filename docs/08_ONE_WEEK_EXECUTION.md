# One-Week Execution Plan

## Definition of success
By the end of the week:
- application runs on a VPS;
- dashboard is accessible;
- database is live;
- at least three approved job sources ingest correctly;
- scoring works;
- a grounded resume packet is generated;
- approval workflow works;
- interview pack is generated;
- consulting lead flow works;
- automated tests and smoke tests pass;
- Codex audit is complete;
- critical defects are repaired;
- Cowork UAT is executed.

## Day 1 — build foundation
- create private repository;
- run Cursor master prompt;
- scaffold monorepo;
- implement database;
- implement API and dashboard shell;
- implement configuration;
- run baseline tests.

## Day 2 — ingestion and scoring
- public ATS feeds;
- dedicated Gmail interface;
- normalization;
- deduplication;
- salary and location logic;
- risk-adjusted scoring;
- fixtures and tests.

## Day 3 — Career Vault and documents
- evidence registry;
- resume variants;
- DOCX and PDF generation;
- Drive integration;
- application packet;
- approval state machine.

## Day 4 — interview and consulting
- Interview Grilling Machine;
- recruiter-signal flow;
- consulting-lead pipeline;
- dashboard integration.

## Day 5 — cloud and CI/CD
- Hostinger;
- Docker Compose;
- n8n;
- GitHub Actions;
- backups;
- monitoring;
- deployment.

## Day 6 — audit and repair
- Codex audit;
- Claude Code repair;
- security and regression tests;
- redeploy.

## Day 7 — UAT and launch
- Cowork UAT;
- final fixes;
- ChatGPT signoff;
- enable daily schedules;
- begin supervised campaign.

## Warning
This is an aggressive plan. It requires long focused sessions, fast approvals, and no major environment or OAuth blockers.
